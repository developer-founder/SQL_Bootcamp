# 🧠 Relational Data Model and SQL

Проект посвящён изучению реляционной модели данных и практическому применению SQL в PostgreSQL.

В рамках выполнения заданий рассматриваются базовые и продвинутые механизмы работы с данными: работа с представлениями (views), генерация временных рядов, операции над множествами, расчёт производных значений, а также создание и управление материализованными представлениями.

Основная цель проекта — сформировать понимание уровней абстракции в базе данных и научиться отделять логическое представление данных от их физического хранения.

## 🧩 Структура базы данных

Проект основан на реляционной модели, включающей:

- **person** — пользователи системы  
- **pizzeria** — список пиццерий  
- **menu** — меню и цены  
- **person_visits** — посещения пиццерий  
- **person_order** — заказы пользователей  

Связи между таблицами заданы явно через первичные и внешние ключи.

## 🧪 Задание 00

Создай два представления:

- `v_persons_female` — пользователи женского пола  
- `v_persons_male` — пользователи мужского пола  

Представления должны содержать те же атрибуты, что и исходная таблица `person`, с фильтрацией по полу.

---

## 🧪 Задание 01

Используя представления `v_persons_female` и `v_persons_male`, выведи общий список имён.

Требования:
- объединить результаты в один запрос
- отсортировать по имени

---

## 🧪 Задание 02

Создай представление `v_generated_dates`, которое содержит даты:

- с 2022-01-01 по 2022-01-31
- тип данных: DATE
- сортировка по дате

Генерация должна выполняться с использованием `generate_series(...)`.

---

## 🧪 Задание 03

Найди пропущенные даты посещений за январь 2022 года.

Используй:
- представление `v_generated_dates`
- таблицу посещений

Результат должен содержать только:
- `missing_date`

---

## 🧪 Задание 04

Реализуй симметрическую разность множеств:

(R − S) ∪ (S − R)

Где:
- R — посещения 2 января 2022
- S — посещения 6 января 2022

Результат:
- только `person_id`
- сортировка по `person_id`

Сохрани результат в представление `v_symmetric_union`.

---

## 🧪 Задание 05

Создай представление `v_price_with_discount`, которое возвращает:

- имя клиента
- название пиццы
- цену
- цену со скидкой 10%

Дополнительно:
- скидка = price - price * 0.1
- округление до целого числа
- сортировка по имени и названию пиццы

---

## 🧪 Задание 06

Создай материализованное представление `mv_dmitriy_visits_and_eats`, которое содержит:

- пиццерии, которые посетил Дмитрий 2022-01-08
- и где он мог заказать пиццу дешевле 800

Материализованное представление должно быть создано с сохранением данных.

---

## 🧪 Задание 07

Обнови состояние `mv_dmitriy_visits_and_eats`:

- добавь новый визит Дмитрия, удовлетворяющий условиям
- убедись, что результат обновляется через refresh materialized view

---

## 🧪 Задание 08

Удалить из базы данных:

- все созданные представления (views)
- материализованное представление

Цель — полностью очистить созданные объекты проекта.
