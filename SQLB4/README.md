# 🧠 Relational Data Model and SQL

Проект посвящён изучению реляционной модели данных и практическому применению SQL в PostgreSQL.

В рамках выполнения заданий рассматриваются базовые и продвинутые механизмы работы с данными: фильтрация, сортировка, подзапросы, объединения таблиц, работа с множествами и построение аналитических запросов.

Основная цель проекта — сформировать устойчивое понимание SQL как декларативного языка, где данные рассматриваются как множества, а не как последовательные инструкции.

- Используй актуальную версию PostgreSQL

- В репозитории должны оставаться только файлы, указанные в заданиях

## 🧩 Структура базы данных

Проект основан на реляционной модели, включающей:

- **person** — пользователи системы
- **pizzeria** — список пиццерий
- **menu** — меню и цены
- **person_visits** — посещения пиццерий
- **person_order** — заказы пользователей

Связи между таблицами заданы явно через первичные и внешние ключи.

## 🧪 Задание 00 — Let's find appropriate prices for Kate

Получить список:

- названия пицц
- цены
- названия пиццерий
- даты посещений

для клиента Kate с ценами от 800 до 1000 рублей.

Отсортировать по названию пиццы, цене и пиццерии.

## 🧪 Задание 01 — Let's find forgotten menus

Найти все `menu_id`, которые ни разу не были заказаны.

Отсортировать по `menu_id`.

## 🧪 Задание 02 — Let's find forgotten pizza and pizzerias

Используя результат предыдущего задания, вывести:

- название пиццы
- цену
- название пиццерии

для меню, которые никогда не заказывались.

Отсортировать по названию пиццы и цене.

## 🧪 Задание 03 — Let's compare visits

Найти пиццерии, которые посещались чаще женщинами или мужчинами.

Разрешено использовать операции множеств с сохранением дубликатов.

Отсортировать по названию пиццерии.

## 🧪 Задание 04 — Let's compare orders

Найти объединение пиццерий, в которые заказы делали только женщины, и только мужчины.

Использовать операции множеств без дубликатов.

Отсортировать по названию пиццерии.

## 🧪 Задание 05 — Visited but did not make any order

Отсортировать по названию.

## 🧪 Задание 06 — Find price-similarity pizzas

Найти одинаковые пиццы (по названию и цене), но из разных пиццерий.
