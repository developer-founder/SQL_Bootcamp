# 🧠 Day 01 — SQL Bootcamp (Relational Data Model & SQL)

Проект посвящён практическому изучению реляционной модели данных и работе с SQL в PostgreSQL.

В рамках выполнения заданий отрабатываются ключевые навыки: работа с множествами, объединения и пересечения данных, декартово произведение, подзапросы, различные виды JOIN и сравнение подходов IN/EXISTS.

Основная цель — научиться мыслить SQL как языком работы с множествами данных, а не последовательных операций.

## 🧩 Структура базы данных

Проект основан на реляционной модели, включающей таблицы:

- **person** — пользователи системы  
- **pizzeria** — список пиццерий  
- **menu** — доступные блюда и цены  
- **person_visits** — посещения пиццерий  
- **person_order** — заказы пользователей  

Связи между таблицами реализованы через первичные и внешние ключи.

---

## 🧪 Задание 00 — UNION Basics

Объединить данные из таблиц:

- `person` (id, name)  
- `menu` (id, pizza_name)

В один список с едиными полями.

---

## 🧪 Задание 01 — UNION + сортировка

Изменить результат предыдущего задания:
- убрать идентификатор
- отсортировать сначала людей, затем пиццы

---

## 🧪 Задание 02 — Уникальные значения

Получить список уникальных названий пицц:

- без DISTINCT / GROUP BY / JOIN
- сортировка по убыванию

---

## 🧪 Задание 03 — Пересечение данных

Найти пользователей, которые:
- делали заказ
- и посещали пиццерию
- в один и тот же день

Сравниваются даты и `person_id`.

---

## 🧪 Задание 04 — Разность множеств

Вычислить разницу между:
- заказами (`person_order`)
- посещениями (`person_visits`)

Только за 07.01.2022  
С сохранением дубликатов.

---

## 🧪 Задание 05 — Декартово произведение

Сформировать все возможные комбинации:

- `person × pizzeria`

Каждый человек должен быть сопоставлен с каждой пиццерией.

---

## 🧪 Задание 06 — Анализ пересечений (обогащение данных)

Модификация задания 03:
- вместо ID вывести имена пользователей
- сортировка: дата ↑, имя ↓

---

## 🧪 Задание 07 — JOIN + форматирование

Получить:
- дату заказа
- строку вида: `Имя (age:XX)`

---

## 🧪 Задание 08 — NATURAL JOIN

Переписать JOIN из задания 07 с использованием:

- `NATURAL JOIN`

Результат должен совпадать.

---

## 🧪 Задание 09 — IN vs EXISTS

Найти пиццерии, в которые **не заходили пользователи**:

- решение через `IN`
- решение через `EXISTS`

---

## 🧪 Задание 10 — Полное объединение данных

Получить полную цепочку:

- имя человека
- название пиццы
- название пиццерии

Сортировка по всем полям (ASC).
