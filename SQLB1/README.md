# 🧠 Relational Data Model and SQL

Проект посвящён изучению реляционной модели данных и практическому применению SQL в PostgreSQL.

В рамках выполнения заданий рассматриваются базовые и продвинутые механизмы работы с данными: фильтрация, сортировка, подзапросы, объединения таблиц, работа с множествами и построение аналитических запросов.

Основная цель проекта — сформировать устойчивое понимание SQL как декларативного языка, где данные рассматриваются как множества, а не как последовательные инструкции.

## ⚙️ Рекомендации к выполнению проекта

- Используй актуальную версию PostgreSQL
- Пиши SQL-запросы в удобной среде разработки (DataGrip, DBeaver, pgAdmin и др.)
- В репозитории должны оставаться только файлы, указанные в заданиях
- Настрой `.gitignore`, чтобы исключить лишние файлы
- Перед началом работы разверни предоставленную модель базы данных
- Внимательно соблюдай ограничения каждого задания
- Проверяй не только синтаксис, но и логическую корректность результата

## 🧩 Структура базы данных

Проект основан на реляционной модели, включающей:

- **person** — пользователи системы  
- **pizzeria** — список пиццерий  
- **menu** — меню и цены  
- **person_visits** — посещения пиццерий  
- **person_order** — заказы пользователей  

Связи между таблицами заданы явно через первичные и внешние ключи.

---

## 🧪 Задание 00

Получить список людей, проживающих в городе «Казань»:

- имя  
- возраст  

---

## 🧪 Задание 01

Вывести имена и возраст женщин из города «Казань».  
Результат отсортировать по имени.

---

## 🧪 Задание 02

Реализовать два запроса, возвращающих пиццерии с рейтингом от 3.5 до 5:

- с использованием операторов сравнения (`>=`, `<=`)
- с использованием `BETWEEN`

---

## 🧪 Задание 03

Найти уникальные ID пользователей, которые:
- посещали пиццерии с 6 по 9 января 2022 года
- либо посещали пиццерию с ID = 2

Результат отсортировать по убыванию ID.

---

## 🧪 Задание 04

Сформировать строку вида:

`Name (age:X,gender:'Y',address:'Z')`

и отсортировать результат по этой строке.

---

## 🧪 Задание 05

Получить имена пользователей, которые сделали заказы на позиции меню (13, 14, 18) 7 января 2022 года.

Использование `IN` и `JOIN` запрещено — только подзапросы.

---

## 🧪 Задание 06

Расширение задания 05:
- добавить вычисляемый столбец `check_name`
- если имя = 'Denis' → `true`, иначе `false`

---

## 🧪 Задание 07

Разделить пользователей по возрастным интервалам:

- 10–20 → interval #1  
- 21–23 → interval #2  
- остальные → interval #3  

Отсортировать по интервалу.

---

## 🧪 Задание 08

Вывести все записи из `person_order`, где `id` — чётное число.

---

## 🧪 Задание 09

Получить:
- имя человека
- название пиццерии

из посещений за период 7–9 января 2022 года

через вложенные подзапросы в `SELECT` и `FROM`.
