# 🧠 Relational Data Model and SQL

Проект посвящён изучению реляционной модели данных и практическому применению SQL в PostgreSQL.

В рамках выполнения заданий рассматриваются базовые и продвинутые механизмы работы с данными: фильтрация, сортировка, подзапросы, объединения таблиц, работа с множествами и построение аналитических запросов.

Основная цель проекта — сформировать устойчивое понимание SQL как декларативного языка, где данные рассматриваются как множества, а не как последовательные инструкции.

## ⚙️ Рекомендации к выполнению проекта

- Используй актуальную версию PostgreSQL
- Пиши SQL-запросы в удобной среде разработки (DataGrip, DBeaver, pgAdmin и др.)
- В репозитории должны оставаться только файлы, указанные в заданиях
- Настрой `.gitignore`, чтобы исключить лишние файлы
- Перед началом работы разверни предоставленную модель базы данных
- Внимательно соблюдай ограничения каждого задания
- Проверяй не только синтаксис, но и логическую корректность результата

## 🧩 Структура базы данных

Проект основан на реляционной модели, включающей:

- **person** — пользователи системы  
- **pizzeria** — список пиццерий  
- **menu** — меню и цены  
- **person_visits** — посещения пиццерий  
- **person_order** — заказы пользователей  

Связи между таблицами заданы через первичные и внешние ключи.

## 🧪 Задание 00

Вывести список пиццерий с рейтингом, которые никто не посещал.

## 🧪 Задание 01

Найти отсутствующие даты посещений с 1 по 10 января 2022 года для пользователей с ID 1 и 2.

## 🧪 Задание 02

Получить:

- список людей, которые посещали или не посещали пиццерии с 1 по 3 января 2022 года  
- список пиццерий, которые посещались или не посещались в этот же период  

NULL значения заменить на `'-'`. Результат отсортировать по всем колонкам.
